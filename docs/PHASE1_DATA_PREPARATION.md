## 核心动机与原理 (The "Why")

### 1. 为什么需要对称归一化邻接矩阵？

**原始问题：** 在图卷积中，如果我们直接用原始邻接矩阵 $A$ 进行消息传播：
$$H^{(l+1)} = A H^{(l)} W^{(l)}$$

会出现以下问题：
- **度数的指数级放大**：高度数的节点会积累越来越多的特征值，导致数值不稳定
- **梯度消失/爆炸**：多层传播后，信号强度随节点度数剧烈波动

**归一化的直觉理解：**
想象一个学术引用网络：
- 一篇被引用1000次的综述论文（高度数节点）
- 一篇被引用5次的专业论文（低度数节点）

如果不归一化，综述论文会"淹没"其他节点的信号。归一化让每个节点平等地聚合邻居信息，无论它有多少邻居。

**数学原理：**
对称归一化公式：$\hat{A} = D^{-1/2} \tilde{A} D^{-1/2}$

其中：
- $\tilde{A} = A + I$（添加自环，让节点考虑自身特征）
- $D_{ii} = \sum_j \tilde{A}_{ij}$（度矩阵）
- $D^{-1/2}$ 实现了"按连接数平分信号"的效果

这本质上是一种**谱归一化**，让图卷积算子的特征值在 $[-1, 1]$ 范围内，保证多层传播的数值稳定性。

---

### 2. 为什么要用稠密张量（Dense Tensor）？

**背景：** 2708×2708 的邻接矩阵只有 5429 条边，稀疏度为 0.07%，是典型的稀疏矩阵。

**选择稠密张量的原因：**

1. **教学清晰性**：从零实现时，稠密矩阵的 `torch.matmul(adj, x)` 直观易懂；稀疏矩阵需要理解 `torch.sparse.mm` 和 COO/CSR 格式，增加认知负担。

2. **Cora 规模可控**：2708×2708 的 float32 张量仅需约 29MB 内存，现代 GPU 完全可容纳。

3. **算法验证优先**：在确保算法正确性之前，过早优化会掩盖潜在 bug。

** trade-off：**
- 优势：代码可读性强，调试方便
- 代价：PubMed（19K节点）数据集会占用约 1.5GB，需要后续扩展为稀疏实现

---

### 3. 为什么特征需要 L2 归一化？

**原因：** 词袋特征（Bag-of-Words）的数值分布高度不均匀。

- 某些高频词可能出现上百次
- 某些专业术语只出现几次

如果不归一化，模型会被高频词支配。L2 归一化让每个文档的特征向量模长为1，消除词频差异的影响，让模型关注**词的分布模式**而非**绝对词频**。

---

### 4. 为什么采用 140/500/1000 的数据划分？

这是**半监督学习**的经典设定：

- **140个训练节点**（每类20个）：模拟"只有少量标签"的真实场景
- **500个验证节点**：用于早停（Early Stopping）和超参数调优
- **1000个测试节点**：评估模型泛化能力
- **1068个无标签节点**：训练时可以使用这些节点的特征和连接信息，但不使用它们的标签

这种设定体现了 GCN 的核心优势：**利用图结构传播有限标签信息到整个图**。

## 实现方式 (Methodology)

### 归一化实现细节

```python
# Step 1: 添加自环
A_tilde = A + I

# Step 2: 计算度矩阵
D = torch.sum(A_tilde, dim=1)

# Step 3: 计算 D^{-1/2}
d_inv_sqrt = torch.pow(D, -0.5)

# Step 4: 对称归一化（使用广播机制）
A_hat = d_inv_sqrt.view(-1, 1) * A_tilde * d_inv_sqrt.view(1, -1)
```

**关键技巧：** 利用 PyTorch 广播避免构造对角矩阵，节省内存。

### 数据流架构

```
cora.content  cora.cites
     ↓              ↓
[Parse Content]  [Parse Cites]
     ↓              ↓
  X [2708, 1433]  edge_list
     ↓              ↓
[L2 Normalize]   [Dense Adj]
     ↓              ↓
  X_norm          A [2708, 2708]
     ↓              ↓
     └──────┬──────┘
            ↓
    [Symmetric Normalize]
            ↓
    A_hat [2708, 2708]
            ↓
    [Create Masks]
            ↓
    train_mask [2708] ← 140 True
    val_mask   [2708] ← 500 True
    test_mask  [2708] ← 1000 True
```
